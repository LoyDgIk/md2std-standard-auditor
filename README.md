# md2std Standard Auditor

`md2std-standard-auditor` 是独立的中文标准 Markdown 审校器。它不生成 Word，不替代 `md2std` 排版生成器，只在转换前检查 `md2std` Markdown 是否违反基础审校规则。

## 当前校验范围

- 禁止旧交叉引用语法 `{@...}`。
- 禁止在标题、表题、图题、公式中手写生成编号。
- 禁止 LaTeX `\tag{...}`。
- 校验 `{{tbl:...}}`、`{{fig:...}}`、`{{eq:...}}`、`{{std:...}}` 语法。
- 校验表、图、公式锚点类型和 ID 格式。
- 校验表、图、公式锚点重复和未知引用。
- 校验 `{{std:...}}` 是否能在“规范性引用文件”章中找到对应标准号。
- 理解当前 md2std 标准 Markdown 契约：显式术语块 `{术语：...}`、无标题条 `{无标题条:n}`、组合图题 `{图：#fig:id}`、图表通用附加项 `{单位}`/`{来源}`/`{脚注}`、图标引 `{图标引}`、图段 `{图段}`、分图组 `{分图组:n}`、表内注 `〔注：...〕` 和表内脚注引用 `〔脚注〕`。
- 禁止已移除的旧式图表附加项和旧分图语法，例如 `{表注：...}`、`{图注：...}`、`{表单位：...}`、`{图脚注a：...}`、`{图标引1：...}`、`{图段：...}`、`{分图：...}`。
- 检查图表通用附加项是否紧跟合法表格或图片目标，并检查同一目标下 `{单位}`、`{来源}` 是否重复。
- 检查 `# 范围` 是否存在并位于首个一级章。
- 检查附录、参考文献、索引的基本顺序。
- 检查附录中是否错误设置“范围”“规范性引用文件”“术语和定义”等章。
- 检查规范性引用文件章是否错误分条、使用列表符号、缺少固定导语或与“无引用”说明冲突。
- 检查规范性引用文件清单顺序是否疑似违反标准类型和编号排序规则。
- 禁止在参考文献条目中手写 `[1]`、`[2]` 等方括号序号；编号由 md2std/Word 参考文献样式生成。
- 提醒标准编号年份连接号疑似误用 `-` 或 `–` 的情况，应使用 `—`。
- 提醒参考文献中机械列入 GB/T 1.1 的情况。
- 提醒资料性附录中疑似出现要求、推荐或禁止类能愿动词。
- 提醒条文注、图中的注、表中的注等注中疑似出现要求、推荐或禁止类能愿动词。
- 检查索引字母分组顺序，以及索引项是否疑似归入错误拼音首字母分组。
- 检查表题后是否紧跟表格，以及表格是否缺少 `{表：#tbl:id}` 表题。
- 检查块级公式是否缺少 `#eq:id` 锚点。
- 提醒列项细分疑似超过两个层次。
- 提醒缺少 YAML front matter。

这些规则最初迁移自旧 `md2std-standard-skill` 的 validation checklist，后续审校规则以本项目为唯一维护位置。更完整的 GB/T 1.1—2020 语义审校可以在后续版本继续扩展。

## 使用

开发安装：

```powershell
python -m pip install -e .
```

直接运行模块：

```powershell
python -X utf8 -m md2std_standard_auditor input.md
```

安装后使用命令：

```powershell
md2std-audit input.md
```

默认只在 `error` 级别返回非零退出码。如需让 warning 也失败：

```powershell
md2std-audit input.md --fail-level warning
```

输出 JSON：

```powershell
md2std-audit input.md --format json
```

## 与 md2std 的边界

- `md2std`：Markdown -> Word 排版生成器。
- `md2std-standard-auditor`：Markdown -> 审校报告。
- `standard-audit-workflow`：编排机械校验和 AI 清单审查，解释审校结果。
- `standard-drafting-workflow`：编排完整标准编写流程，并将审校任务委托给审校 skill。
- `md2std-standard-skill`：面向 Codex 的生成器输入说明和 DOCX 生成包装，不承载审校流程。
